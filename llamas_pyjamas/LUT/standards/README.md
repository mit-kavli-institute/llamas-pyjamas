# Spectrophotometric standard stars

Reference data for identifying and flux-calibrating standard-star exposures. LLAMAS headers do
not flag standards — they carry `OBS TYPE = SCIENCE` like any target — so they are found by
crossmatching the pointing against the coordinates here.

Generated by `python -m llamas_pyjamas.Scripts.fetch_standards`; re-run to refresh.

## Contents

- **`index.dat`** — one row per standard: `name`, `ra_deg`, `dec_deg` (decimal J2000), `vmag`,
  `sptype`, `flux_file`. 68 stars from the ESO
  [RA-ordered list of spectrophotometric standards](https://www.eso.org/sci/observing/tools/standards/spectra/stanlis.html).
- **`flux/f<name>.dat`** — Oke (1990) reference spectra, from the STScI CALOBS set mirrored at
  ESO/STECF. Four columns: wavelength (Å), flux (10⁻¹⁶ erg cm⁻² s⁻¹ Å⁻¹), flux (mJy), bin (Å).
  See `flux/aaareadme.oke`.

## Coverage

Coordinates cover all 68 ESO optical standards. Reference spectra are bundled for the 25 that
appear in the Oke set — including both standards in the may26 commissioning night, **Feige110**
and **GD108**. The remaining stars (bright HR / southern CTIO standards) have coordinates but no
bundled spectrum; their flux tables live in ESO's `ctiostan/`, `hststan/`, `wdstan/` archives,
whose file naming is inconsistent enough that they are left out rather than risk mapping a star
to the wrong spectrum. Extend `fetch_standards.py` if a future night needs one of them.

## Consumed by

`llamas_pyjamas.Flux.fluxStandards` loads `index.dat` and crossmatches pointings (default 30″).
